# Hybrid PET reconstruction

## Goal

Define a hybrid workflow that combines:

1. partial inference from a target integer under bounded factorization budgets
2. forward constructive PET synthesis from explicit local update histories

The aim is not to recover the exact PET of an arbitrary large target without factorization.
The aim is to extract reliable partial structural constraints from the target, then build forward PET-compatible candidate integers that satisfy those constraints.

In short:

- **pure inverse reconstruction** is too strong
- **pure forward construction** is controllable but target-agnostic
- **hybrid reconstruction** tries to bridge the two

## Existing ingredients already present in the repo

### 1. Partial bounded probe of a target integer

Tool:

- `tools/partial_signature_probe.py`

This tool already provides a budgeted inspection workflow based on:

- bounded `factorint(...)`
- residual splitting into known prime factors and unresolved remainder
- residual classification
- optional `pollard_rho` fallback
- recursive schedule-driven probing

Its final report already exposes several useful fields:

- `known_factors`
- `known_root_children`
- `known_root_generator_lower_bound`
- `residual`
- `residual_info`
- `root_generator_lower_bound`
- `exact_root_anatomy`
- `exact_root_children`
- `exact_root_generator`
- `fully_factored`
- `stop_reason`
- `closure_kind`

So this tool is already a viable **Phase A** for a hybrid workflow.

### 2. Forward constructive PET plans

Tool:

- `tools/pet_constructive_plan.py`

This tool builds integers directly from local PET update plans using:

- `attach p`
- `bump p`

It validates each local step and exposes:

- final integer
- generator
- signature
- local branch evolution

So this tool is already a viable **Phase C** engine for constructive synthesis.

### 3. Local exponent dynamics

Tool:

- `tools/exponent_shape_trace.py`

This tool studies exponent-shape traces over exponent ranges and helps understand the local structural dynamics induced by repeated `bump` operations.

It is useful as an auxiliary analysis tool for choosing plausible constructive completions.

## Hybrid pipeline

The hybrid reconstruction workflow can be described in four phases.

### Phase A — bounded target probe

Input:

- a target integer `N`
- a schedule of bounded factorization budgets

Action:

- run `tools/partial_signature_probe.py`
- collect only the partial information that is certified or clearly justified by the probe

Output:

- known prime factors and exponents
- a residual cofactor
- residual classification
- lower bounds on PET root anatomy
- exact root anatomy only if the probe actually closes it

This phase does **not** claim exact reconstruction in general.

### Phase B — bridge to constructive constraints

Translate the probe output into a constructive payload.

This is the key bridge between inference and synthesis.

The bridge payload should separate:

- **hard constraints**: things directly supported by the probe
- **soft degrees of freedom**: unresolved parts of the target

Examples of hard constraints:

- prime `2` appears with exponent `5`
- prime `3` appears with exponent `1`
- the residual contributes at least two additional root children
- the root generator lower bound is at least `12`

Examples of unresolved freedom:

- how the unknown residual should be completed
- how many extra branches beyond the lower bound should be introduced
- whether the unresolved mass should be modelled as new attaches, deeper bumps, or both

This phase does **not** yet construct any integer.
It produces a partial constructive specification.

### Phase C — forward synthesis under constraints

Use the constructive specification to build one or more candidate integers `M`.

Each candidate should:

- include all known factors extracted in Phase A
- respect all hard structural lower bounds
- remain explicit about where the unknown residual is being modelled rather than recovered
- be generated by a valid forward PET history

This phase is inherently generative, not inferential.
It does not say “this is the target”.
It says “this is a forward PET-compatible candidate explanation of the target, given the current probe information”.

### Phase D — candidate comparison and ranking

Compare candidate integers against what is actually known about the target.

Possible ranking criteria include:

- satisfaction of all hard constraints
- simplicity of the constructive history
- closeness to lower bounds rather than gratuitous complexity
- compatibility with any exact root anatomy already certified by the probe
- consistency with additional arithmetic metadata, if available

The output of this phase is not “the true decomposition”.
It is a ranked family of candidate PET-guided explanations.

## Minimal bridge payload

A minimal payload passed from Phase A to Phase B could look like this:

```json
{
  "target": "N",
  "known_factors": [
    {"prime": 2, "exp": 5},
    {"prime": 3, "exp": 1}
  ],
  "residual": "R",
  "residual_info": {
    "status": "composite-non-prime-power",
    "root_children_lower_bound": 2
  },
  "known_root_generator_lower_bound": 6,
  "root_generator_lower_bound": 12,
  "exact_root_anatomy": false
}
```

This should be read as:

- part of the target is already known exactly
- the unresolved residual still contributes structural mass
- the final candidate must at least respect those lower bounds
- any completion beyond that is modelling, not certified recovery

## Reading `partial_signature_probe` as Phase A output

The current probe report can already be interpreted almost directly as a hybrid input.

### Certified components

The following are suitable as hard constraints:

- `known_factors`
- `known_root_children`
- `known_root_generator_lower_bound`
- `root_generator_lower_bound`
- `exact_root_children`
- `exact_root_generator`
- `fully_factored`

### Residual-side uncertainty

The following fields capture uncertainty and should remain soft or conditional:

- `residual`
- `residual_info`
- `stop_reason`
- `closure_kind`
- `last_progress_stage_kind`

This distinction is important.

The hybrid workflow should never silently promote:

- lower bounds into exact equalities
- stalled residuals into invented structure
- composite unknowns into fake prime explanations

## What the synthesis side should do

A minimal synthesis strategy could be:

1. materialize the known factors as an initial partial root anatomy
2. treat the residual lower bound as the minimum unexplained root contribution
3. generate one or more completions with explicit modelling choices
4. score those completions instead of pretending they are exact inversions

In this sense, the synthesis side is not solving factorization.
It is building explanatory forward candidates under explicit uncertainty.

## Example interpretation

Suppose the probe yields:

- known factors: `2^5 * 3`
- residual status: composite non-prime-power
- residual root children lower bound: `2`
- root generator lower bound: `12`

Then a legitimate hybrid reading is:

- the target definitely contains a branch for `2` with exponent `5`
- the target definitely contains a branch for `3` with exponent `1`
- the unresolved residual contributes at least two more root children
- any candidate completion must therefore contain at least four root children in total, unless some exact closure information later overrides that interpretation

At this point, a synthesis engine may produce several constructive candidates compatible with those conditions.
That is acceptable.
Claiming that any one candidate is “the recovered target PET” would not be acceptable.

## Non-claims

This workflow does **not** claim any of the following in general:

- exact PET recovery from an arbitrary large target without factorization
- replacement of integer factorization by PET reasoning
- unique reconstruction from partial metadata
- exact inversion in the non-coprime case

These remain out of reach.

## Provisional claim

What this workflow *can* claim is much narrower and much more honest:

> From a difficult target integer, we may be able to extract reliable partial structural constraints under bounded factorization effort, and then construct forward PET-compatible candidate explanations that satisfy those constraints.

This is not exact inversion.
It is constrained structural reconstruction under explicit uncertainty.

## Worked example: a `perfect-power-composite-base` residual

A useful hybrid example is provided by the target

- `N = 212262408`
- construction label: `2*(101*102)^2`

Using a bounded probe with a very small schedule, this target does **not** close exactly, but its unresolved residual is still classified in a more informative way than a generic composite unknown.

### Phase A — probe

A search with `tools/find_probe_status_examples.py` produced this target as an example with:

- `residual_info.status = "perfect-power-composite-base"`
- `fully_factored = false`
- `exact_root_anatomy = false`

The probe also extracted certified known factors:

- `2^3`
- `3^2`

So the hybrid workflow begins from a situation where:

- part of the target is known exactly
- the residual is not closed
- the residual is nevertheless recognized as a perfect power with composite base

This is more structured than the generic status:

- `composite-non-prime-power`

and therefore provides a better stress test for the bridge and synthesis stages.

### Phase B — bridge interpretation

For this target, the bridge payload separates:

- hard constraints coming from the known factors
- soft uncertainty attached to the unresolved perfect-power composite residual

The important point is that the workflow still does **not** claim to have recovered the exact hidden factorization of that residual.

Instead, it records that the unresolved mass has a more structured shape than an arbitrary composite residue.

### Phase C — synthesis and ranking

The current synthesizer still applies generic completion strategies rather than a status-specific strategy for `perfect-power-composite-base`.

That is acceptable for now.

The important fact is that the hybrid pipeline remains operational:

1. the probe extracts partial certified structure
2. the bridge preserves uncertainty without collapsing it into fake exactness
3. the synthesizer produces ranked forward-compatible candidates
4. the target can be studied as a constrained reconstruction problem rather than an all-or-nothing inversion problem

### Why this example matters

This example shows that the hybrid workflow is not limited to only two trivial extremes:

- targets that fully close under the probe
- targets with only a generic unresolved composite residue

It also works on an intermediate case where the residual is still unresolved but already carries a recognizable structural class.

That is an important milestone because it shows that the pipeline can react not only to “known” versus “unknown”, but also to *different kinds of unknown*.

## Updated status

This is no longer only a research specification.

Relevant tools now present in the repo:

- `tools/partial_signature_probe.py`
- `tools/pet_hybrid_bridge.py`
- `tools/pet_hybrid_synthesize.py`
- `tools/find_probe_status_examples.py`
- `tools/pet_constructive_plan.py`
- `tools/exponent_shape_trace.py`

So the current state is:

- bounded probe of target integers
- machine-readable bridge payload
- forward candidate synthesis
- candidate ranking
- targeted search for examples by residual status

What still does **not** exist is a status-specialized synthesizer.
For example, `perfect-power-composite-base` is currently handled by the same generic completion strategies used for other unresolved residual types.
